# CLAUDE.md

## Development principles

1. **KISS**
   Prefer the simplest solution that works.

2. **YAGNI**
   Do not add infrastructure, abstractions, or features before they are needed.

3. **TDD**
   When implementation begins, define expected behavior with tests or validation checks.

4. **DRY**
   Keep schemas, rules, and workflow definitions in one canonical place.

5. **SOLID**
   Keep responsibilities narrow and interfaces clear.

## Working rules

- Make the smallest correct change.
- Do not add speculative features.
- Prefer editing existing files over creating new ones.
- Keep documentation canonical and non-duplicative.
- Preserve source anchors and stable identifiers where possible.
- Keep accepted knowledge content separate from mutable review state.
- Do not treat experiments as part of the mandatory daily loop.
